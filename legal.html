Privacy Policy

Last updated: March 25, 2026

weejur ("the Service") is a personal project operated by npilk ("I", "me"). This policy explains what information the Service accesses, collects, and how it's used.

The short version: weejur collects as little data as possible. It uses GitHub OAuth so you can publish to your own account, and it uses privacy-friendly analytics so I can understand how the site is used. I don't sell data, I don't run ads, and I don't use tracking cookies.

Information Collected Through GitHub OAuth

When you sign in, the Service uses GitHub's OAuth flow to authenticate you. This grants the Service access to:

Scope What it accesses Why
public_repo Your public repositories To create, update, and manage the GitHub Pages repositories for your published sites
delete_repo Ability to delete repositories To let you delete sites you've published through the dashboard

The Service accesses your GitHub account only to perform the actions you request (publishing, updating, or deleting sites). Your GitHub OAuth token is stored in your browser's localStorage and never sent to or stored on any server controlled by weejur. The token only leaves your browser when making API calls directly to GitHub.

You can revoke the Service's access at any time in your GitHub authorized applications settings.

For details on how GitHub handles your data, see GitHub's Privacy Statement.

Analytics

The Service uses Umami, a privacy-focused, open-source analytics tool. The Umami instance is self-hosted and served from stats.weejur.com.

Here's what Umami does and doesn't do:

  • No cookies. Umami does not set any cookies in your browser.
  • No personal data. It does not collect IP addresses, fingerprints, or any information that could identify you personally.
  • No cross-site tracking. There is no way to connect your visit here to your activity on any other website.
  • What is collected: Aggregated, anonymous page view counts, referrer information, browser/OS type, and device type. This helps me understand if anyone is actually using this thing and which pages are most useful.

Umami is compliant with GDPR, CCPA, and PECR. No consent banner is required because no personal data is processed and no cookies are used. For more details, see Umami's privacy information.

Cookies and Local Storage

The Service does not use cookies. It uses your browser's localStorage to store:

  • Your GitHub OAuth access token (so you stay logged in between visits)
  • Your GitHub username (to display in the UI)

This data never leaves your browser except when making authenticated requests directly to GitHub's API. You can clear it at any time by signing out or clearing your browser's site data.

The Service also uses IndexedDB (browser-local storage) temporarily to hold files you've selected for upload until they are published. These are never sent to any server other than GitHub.

Data Not Collected

To be explicit about what the Service does not do:

  • The Service does not store copies of the content you publish (it goes directly from your browser to your GitHub account via GitHub's API).
  • The Service does not sell, share, or provide any data to third parties.
  • The Service does not run ads or any advertising trackers.
  • The Service does not use any third-party analytics services, social media pixels, or tracking scripts (only the first-party-hosted Umami instance described above).
  • The Service does not have a database or backend that stores user information. The only server-side component is a Cloudflare Worker that exchanges your OAuth code for a token during sign-in — it does not log or retain this data.

Data Retention

Since the Service has no server-side storage, there is no user data to retain. Your authentication token and username exist only in your browser's localStorage until you sign out or clear your browser data. Analytics data is retained in aggregate indefinitely, but since it contains no personal information, it cannot be linked to you.

Content you publish lives in your own GitHub repository. If you want to delete it, you can do so through the weejur dashboard or directly through GitHub. Revoking OAuth access removes the Service's ability to interact with your account.

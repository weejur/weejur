Learn the basics

What is GitHub?

Think of GitHub as a big, free online folder where people store their projects. It's mostly used by programmers, but anyone can use it.

When you use weejur, we put your website files into one of these folders on GitHub. That's it — GitHub keeps them safe and makes them available to the world.

You need a GitHub account to use weejur, but you don't need to know how GitHub works beyond signing in. weejur handles the rest.

What is a repository?

A repository (or "repo") is just a project folder on GitHub. When you create a site with weejur, we create a repo for you with your website files inside it.

Each site you make gets its own repo. If you want to see your files, you can visit your repo on GitHub — you'll find a link on your dashboard.

You don't need to do anything with the repo directly. But it's good to know it exists, because that's where your website lives.

What is Git?

Git is a tool that keeps track of every change you make to your files — like a super-powered "undo" history. GitHub is a website built on top of Git.

You'll sometimes hear people talk about "commits" (saving a snapshot of your changes) or "pushing" (uploading changes). weejur does all of this for you behind the scenes.

You don't need to install Git or learn any commands to use weejur. We mention it here just so you know the word if you see it around.

What is GitHub Pages?

GitHub Pages is a free feature from GitHub that turns a repository into a live website. Instead of just storing your files, GitHub also serves them to anyone who visits your URL.

When you publish a site with weejur, we flip this switch on for you. Your site gets a URL like username.github.io/my-site and it's live on the internet — for free, forever (as long as your GitHub account exists).

There are no ads, no storage limits for normal websites, and no monthly fees. GitHub Pages is genuinely free.

Troubleshooting

Why isn't my site showing up?

Give it a minute. When you first publish, GitHub Pages needs a little time to set up. If you're seeing a 404 error, try waiting a minute or two and then refreshing the page.

Check the URL. Make sure you're visiting the right address. You can find it on your dashboard — click the link there to make sure.

Make sure you have an index.html. GitHub Pages looks for a file called index.html as your home page. If you uploaded files, make sure one of them is named index.html and re-publish.

Still not working? Try deleting the site from your dashboard and creating it again. Sometimes a fresh start fixes things.

How do I get images to show up on my website?

Upload them with your site. If your HTML references images (like <img src="photo.jpg">), you need to upload those image files alongside your HTML in order for them to show up on your website. Use the "Upload files" tab and select multiple files or an entire project folder.

Check file names and extensions. File names are case-sensitive. If your HTML says photo.jpg but the file is called Photo.jpg or photo.png, it won't work. Make sure they match exactly.

Using AI-generated HTML? If you pasted HTML from ChatGPT or Claude, the images in the code might be placeholder URLs that don't actually exist. You'll need to replace them with your own images, or use a service like Unsplash for free stock photos.

How do I use my own domain name?

By default, your website will live at username.github.io/site-name. If you want a custom domain like example.com, it's easy to set up, but it takes a few extra steps (and the domain name will cost a bit of money). GitHub has a detailed guide you can follow, but here's the gist:

1. Buy a domain. You'll need to purchase a domain name from a registrar like Namecheap, Cloudflare, or Porkbun. Prices are often around $10/year.

2. Point it to GitHub. In your domain registrar's DNS settings, add a CNAME record pointing to username.github.io. Your registrar will have instructions, and GitHub's guide has more details as well.

3. Tell GitHub. Go to your repository on GitHub, then Settings → Pages → Custom domain. Enter your domain name and save.

How do I update my site?

Go to your dashboard, find the site you want to update, and click Update. Upload new files or paste updated HTML, then hit Publish again. Your changes will go live within a minute or two at the same URL.

Tip: If you're making small changes, you can also edit files directly on GitHub. Click the repo link on your dashboard, find the file, and click the pencil icon to edit it right in your browser.
